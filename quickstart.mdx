GoodBoyDoc requires only Python 3.7 or later and a pip install. Once installed, you pass your source code as a string to `generate`, then pass the result to `format_output` to get Markdown, HTML, or RST. The steps below walk you through installation and your first documentation run.

## Installation

<Steps>
  <Step title="Install the package">
    Install GoodBoyDoc from PyPI using pip:

    ```bash
    pip install goodboydoc
    ```
  </Step>
  <Step title="Verify the install">
    Confirm the package is available by importing the public API:

    ```python
    from goodboydoc import generate, parse_docstring, format_output
    ```

    If this runs without error, you're ready to go.
  </Step>
</Steps>

## Basic usage

The following example reads a Python source file, generates documentation from all its docstrings, and prints the result as Markdown:

```python
from goodboydoc import generate, format_output

source = open("mymodule.py").read()
docs = generate(source, style="google")
print(format_output(docs, fmt="markdown"))
```

`generate` extracts every triple-quoted docstring from the source string and normalizes it. `format_output` then renders the result in the format you choose.

## Choosing a docstring style

Pass the `style` argument to `generate` or `parse_docstring` to match your project's docstring convention.

<CodeGroup>

```python Google style
from goodboydoc import generate

source = open("mymodule.py").read()
docs = generate(source, style="google")
```

```python NumPy style
from goodboydoc import generate

source = open("mymodule.py").read()
docs = generate(source, style="numpy")
```

```python Sphinx style
from goodboydoc import generate

source = open("mymodule.py").read()
docs = generate(source, style="sphinx")
```

</CodeGroup>

You can also parse a single docstring directly with `parse_docstring`:

```python
from goodboydoc import parse_docstring

raw = """
    Summary line.

    Args:
        name: The name to greet.

    Returns:
        A greeting string.
"""

normalized = parse_docstring(raw, style="google")
print(normalized)
```

<Tip>
  The `style` parameter tells GoodBoyDoc how to normalize section markers. Passing the wrong style won't cause an error, but NumPy-style underlines won't be stripped unless you set `style="numpy"`.
</Tip>

## Choosing an output format

Pass the `fmt` argument to `format_output` to control how the documentation is rendered.

<CodeGroup>

```python Markdown
from goodboydoc import format_output

print(format_output(docs, fmt="markdown"))
```

```python HTML
from goodboydoc import format_output

print(format_output(docs, fmt="html"))
# Output: <div class='goodboydoc'><p>...</p></div>
```

```python RST
from goodboydoc import format_output

print(format_output(docs, fmt="rst"))
```

</CodeGroup>

`"markdown"` is the default if you omit `fmt`. HTML output wraps each paragraph in `<p>` tags inside a `<div class='goodboydoc'>` container. RST output underlines each non-indented line with `=` characters.
